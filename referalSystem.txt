1. CURRENT BUSINESS FLOW (VERY IMPORTANT)
   ==================================================

ONEERP currently DOES NOT use online payment gateways.

Current workflow:

1. User requests activation of a module
2. Admin manually reviews request
3. User pays outside the system
4. Admin manually confirms payment
5. Admin activates module for tenant
6. Tenant receives module access

IMPORTANT:

* Keep this workflow exactly as it is.
* Do NOT replace it with online payment processing.
* Extend the workflow carefully with:

  * referral tracking
  * reward tracking
  * discount calculations
  * activation/payment confirmation statuses

The system architecture should remain scalable so online payments can easily be added in future.
